# Exp - 3 RAILWAY MANAGEMENT SYSTEM

# AIM:

To design and implement a Railway Management System that manages passenger information, train schedules, ticket booking, and staff administration efficiently.

# SRS (Procedure):
# 1. Requirements Specification
Functional Requirements:

User registration and login for passengers.

Booking, cancelling, and modifying train tickets.

Viewing train schedules and seat availability.

Admin management of train details and staff.

Notification system for delays or cancellations.

Non-Functional Requirements:

System should be reliable and available 24/7.

Secure handling of user data and payment information.

User-friendly interface for passengers and staff.

Quick response time for queries and ticket bookings.

# 2. Procedure

Passenger Registration:

Passengers provide personal details and create an account.

Booking Tickets:

Passenger searches train availability.

Selects train, date, and class.

Makes payment and receives ticket confirmation.

Cancellation/Modification:

Passenger requests cancellation or modification.

Refunds processed according to rules.

Admin Functions:

Add/update train schedules.

Assign staff to trains.

Generate reports on bookings and revenue.

Staff Interaction:

Staff manages train operations.

Handles passenger queries and boarding assistance.
